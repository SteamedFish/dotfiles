# Claudeception

## 概述

Claudeception 是一个“用于产出技能的技能”。它会在任务完成后提炼可复用经验，沉淀为新技能，避免后续会话重复踩坑。

## 何时使用

出现以下任一情况时使用：
- 修复过程依赖了非直觉排查或多轮试错。
- 报错信息有误导性，真实根因并不直观。
- 发现了项目特有但未文档化的约定。
- 找到了可重复复用的流程优化。
- 用户明确要求“把这次经验保存成 skill”或做复盘。

以下情况不要使用：
- 纯文档查阅即可解决的问题。
- 仅对一次性场景有效的临时补丁。

## 提取流程

### 1) 先查重

```sh
SKILL_DIRS=(
  ".config/agents/skills"
  "$HOME/.config/agents/skills"
  "$HOME/.codex/skills"
)

rg --files -g 'SKILL.md' "${SKILL_DIRS[@]}" 2>/dev/null
rg -i "keyword1|keyword2" "${SKILL_DIRS[@]}" 2>/dev/null
rg -F "exact error message" "${SKILL_DIRS[@]}" 2>/dev/null
```

决策规则：
- 无相关技能：新建。
- 同触发条件且同修复方案：更新已有技能。
- 同触发条件但根因不同：新建并互相交叉引用。
- 部分重叠：在已有技能中新增变体小节。

### 2) 提炼知识

必须写清：
- 问题与触发条件。
- 非直觉根因。
- 已验证可用的修复步骤。
- 验证方法与成功信号。

### 3) 必要时校验时效性

涉及技术生态最佳实践时，先查官方文档或一手资料。
纯项目内部约定可不联网检索。

### 4) 落地技能

新技能路径：`.config/agents/skills/<skill-name>/SKILL.md`
内容至少包含：
- frontmatter: `name`、`description`
- 问题、触发条件、解决方案、验证方式、注意事项
- 可搜索、可触发的描述文案

### 5) 质量门禁

必须同时满足：
- 可复用，不是一次性经验。
- 非平凡，不是显而易见结论。
- 触发条件具体、步骤可执行。
- 已在实践中验证。
- 不包含密钥或内部敏感信息。

## 复盘模式

每次有价值会话结束后：
1. 回顾本次学习。
2. 列出可提取候选。
3. 优先高复用价值项。
4. 提取 1-3 个技能。
5. 总结新技能及其价值。

## 速查清单

- [ ] 已检索并排除重复技能。
- [ ] `description` 含具体触发条件。
- [ ] 方案有验证证据。
- [ ] 内容可复用且非平凡。
- [ ] 无敏感信息。
- [ ] 涉及技术结论时已附参考来源（可选）。
