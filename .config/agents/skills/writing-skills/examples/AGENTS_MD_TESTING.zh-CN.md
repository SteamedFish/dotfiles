## 要测试的文档变体

### NULL（基线 - 无技能文档）
AGENTS.md 中完全没有提及技能。

### 变体 A：柔和建议
```markdown
## 技能库

你可以在 `~/.config/agents/skills/` 访问技能。考虑
在从事任务之前检查相关技能。
```

```markdown
## 技能库

在从事任何任务之前，检查 `~/.config/agents/skills/` 中的
相关技能。当技能存在时你应该使用它们。

浏览：`ls ~/.config/agents/skills/`
搜索：`grep -r "keyword" ~/.config/agents/skills/`
```

### 变体 C：强调风格
```xml

浏览类别：`ls ~/.config/agents/skills/`
```

### 变体 D：面向流程
```markdown

你每个任务的工作流程：

1. **开始前：** 检查相关技能
   - 浏览：`ls ~/.config/agents/skills/`
   - 搜索：`grep -r "symptom" ~/.config/agents/skills/`

技能库防止你重复常见错误。
在你开始之前不检查就是选择重复那些错误。

从这里开始：`skills/using-skills`
```

## 测试协议

对于每个变体：

1. **首先运行 NULL 基线**（无技能文档）
   - 记录代理选择哪个选项
   - 捕获确切的辩解

2. **用相同时景运行变体**
   - 代理是否检查技能？
   - 如果找到，代理是否使用技能？
   - 如果违反，捕获辩解

3. **压力测试** - 添加时间/沉没成本/权威
   - 代理在压力下仍然检查吗？
   - 记录合规性何时崩溃

4. **元测试** - 问代理如何改进文档
   - "你有文档但没有检查。为什么？"
   - "文档如何更清晰？"

## 成功标准

**变体成功如果：**
- 代理主动检查技能
- 代理在行动之前完全阅读技能
- 代理在压力下遵循技能指导
- 代理无法辩解掉合规性

**变体失败如果：**
- 即使没有压力，代理也跳过检查
- 代理"适应概念"而不阅读
- 代理在压力下辩解掉
- 代理将技能视为参考而非要求

## 预期结果

**NULL：** 代理选择最快的路径，无技能意识

**变体 A：** 如果没有压力，代理可能会检查，在压力下跳过

**变体 B：** 代理有时检查，容易辩解掉

**变体 C：** 强合规性但可能感觉太僵化

**变体 D：** 平衡，但更长 - 代理会内化它吗？
